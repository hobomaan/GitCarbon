# Security Policy

## Supported versions

| Version | Supported |
|---|---|
| 0.1.x (latest) | Yes |

## Reporting a vulnerability

If you discover a security vulnerability in GitCarbon, please report it **responsibly**. Do **not** open a public GitHub issue.

### How to report

Send an email to: **info@pittroche.co.uk**

Include:
- A description of the vulnerability
- Steps to reproduce (if applicable)
- The potential impact
- Your suggested fix (if any)

### What to expect

- **Acknowledgement** within 48 hours
- **Status update** within 7 days
- A fix or mitigation will be prioritised based on severity
- You will be credited in the changelog (unless you prefer otherwise)

### Scope

This policy covers:
- The GitCarbon VS Code extension (`CPR.gitcarbon-vscode`)
- This public repository (documentation, templates, assets)

Out of scope:
- Vulnerabilities in VS Code itself (report to [Microsoft](https://msrc.microsoft.com/create-report))
- Vulnerabilities in Git (report to the [Git project](https://git-scm.com/community))

---

Thank you for helping keep GitCarbon and its users safe.
